# Versionnement API

## Objet

Les APIs REST et GraphQL doivent garantir compatibilité ascendante, dépréciation contrôlée et documentation OpenAPI/GraphQL versionnée.

## Format attendu

- Objectif.
- Périmètre.
- Contrats d’entrée/sortie.
- Règles de sécurité.
- Règles de performance.
- Conditions d’erreur.
- Tests obligatoires.
- Critères d’acceptation.

## Application

Ce RFC est normatif pour toute implémentation liée au sujet. Toute divergence doit être approuvée par ADR.
