# C4 Component

## Objectif

Vue composants : modules RSOT (Ressource Source of Truth), IPAM, DCIM, Discovery, Dependency, ITAM, Security, API.

## Principes de représentation

- Les diagrammes doivent être versionnés avec le code documentaire.
- Chaque flux critique doit être traçable vers une exigence.
- Les frontières de confiance doivent être visibles.
- Les dépendances externes doivent être explicitement nommées.
- Les chemins de données sensibles doivent être identifiés.
- Les points de haute disponibilité et de reprise doivent être matérialisés.

## Livrables associés

Les fichiers sources des diagrammes sont disponibles dans `10-Diagrammes/` afin de permettre génération automatique et revue d’architecture.
