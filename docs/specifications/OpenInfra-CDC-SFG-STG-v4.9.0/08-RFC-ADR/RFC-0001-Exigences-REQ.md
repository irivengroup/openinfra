# Convention des exigences REQ

## Objet

Les exigences utilisent un identifiant stable REQ-xxxxx, une priorité N1/N2/N3, un type, une justification, une vérification et un critère d’acceptation.

## Format attendu

- Objectif.
- Périmètre.
- Contrats d’entrée/sortie.
- Règles de sécurité.
- Règles de performance.
- Conditions d’erreur.
- Tests obligatoires.
- Critères d’acceptation.

## Application

Ce RFC est normatif pour toute implémentation liée au sujet. Toute divergence doit être approuvée par ADR.
