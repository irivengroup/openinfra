# OpenInfra v0.29.8 — validation report

## Scope

OpenInfra v0.29.8 is a corrective P06 release built from v0.29.7. It fixes the PostgreSQL HA semantics requested for “cluster quasi synchrone” by aligning the implementation and CDC wording with near-real-time synchronization, not remote-apply synchronous commits. It also fixes the GitHub Actions migration smoke gate that still expected 23 migrations while v0.29.7 introduced migration 0024.

## Delivered changes

- PostgreSQL cluster topology renamed internally to `near-real-time-streaming-cluster`.
- PostgreSQL HA mode renamed to `near-real-time-postgresql-streaming`.
- Generated PostgreSQL configuration now uses local non-blocking commit semantics:
  - `synchronous_commit = 'local'`.
  - no `synchronous_standby_names` generated by default.
  - WAL streaming, replication slots, PITR archive and backup directories remain managed by the installer.
- HA plan exposes:
  - `commit_policy=local_commit_non_blocking`;
  - `max_expected_lag_seconds=5`;
  - deterministic replication slot names.
- CDC v4.8.1 and roadmap v2 wording updated from “réplication quasi synchrone” to “synchronisation quasi temps réel” where the semantic was ambiguous.
- GitHub Actions migration rendering smoke test changed from hard-coded `23` to a dynamic count of SQL migrations under `installers/migrations/postgresql`.
